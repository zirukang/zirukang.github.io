node基础学习笔记1

node day1

1.1 node

  • 什么是node?

    • nodeJs 是一个构建于Chorme v8 javaScript 解析引擎之上的一个运行时环境
    • Chrome V8 开源的
    • Chrome V8 引擎在所有浏览器的 JavaScript 解析引擎中是最快的
1. 客户端和服务器 任何后台语言都是做三件事
  • 接收请求
  • 处理请求
  • 发送响应
2. EcmaScript 是符合javascript 规范的 （if,else,for,function,array,object）
3.node 的特性
  • event-driven:事件驱动
  • non-blocking I/O model:非阻塞IO模型
    • 同步阻塞型 IO，造成后续代码等待
    • 异步非阻塞，相当于找了一个人帮你去干活儿，自己继续忙活自己的事情
      • callback
      • 一般通常的异步调用API往往伴随着一个回调函数来接收返回值
  • lightweight and efficient: 轻量和高效
  • Node 本身知识提供了一些系统级别操作API

  • 开源免费

  • node 的好处：

    • 前后端编程环境统一，大大降低了前后端语言切换的代价
  • node 的不好处：
    +没有后台编程经验，特别容易分不清前后台代码，因为都是js

1.2 node 语法

  • 注意1：文件名不要使用 ‘node’命名
  • 注意2：文件名不要有中文
    • 用户名不要有中文、空格
    • 目录名也同上

  • 注意3：脚本文件编码都使用 utf8

    1. 如何验证是否有node 环境

      • 打开终端，输入 node -v
    • 在 node 中本身就是一个非常轻量级的一个平台
    • 如果想要使用环境特定 API
    • 本身就是一个高度模块化的一个平台
    • 想要进行文件操作，那就加载 fs 核心模块（node 的核心模块）

    1. 读取文件

    2. 写文件

      • 第一个参数指定一个文件路径
        • 如果该路径不存在，则新建
        • 如果已存在，则覆盖
      • 第二个参数指定要写入文件的内容，默认编码就是utf8
    构建http服务器的核心模块步骤
    • 首先要加载node的核心模块（http）
    • 创建一个 http 服务器，得到一个server 实例对象
    • 监听 server 对象的request 请求事件，设置相应的事件回调处理函数
      • 请求处理函数需要接收两个参数，分别是 Request 请求对象和Response 相应对象
      • request 请求对象：可以用来获取当前请求的一些信息，例如请求路径、请求方法、请求报文数据
      • response 相应对象： 可以用来给当前请求发送响应
        • write 方法可以多次写入响应数据
        • 写入响应数据之后，一定要记得结束响应
        • 否则客户端会认为你的数据没有发送完毕，还要等待接收
        • 可以 使用response.end 方法手动结束响应，本次请求响应结束
          • 同时 response.end 还有一层意思是 结束本次数据，同时发送响应数据
    • 调用 server对象的listen 方法，绑定一个端口号，开启服务器
